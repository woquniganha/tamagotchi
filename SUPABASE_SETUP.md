# Supabase 云存档配置指南

## 为什么需要 Supabase？

游戏默认使用**本地用户系统**（数据存储在浏览器本地），无需任何配置即可使用。
如果你希望实现**跨设备同步**、**云端备份**等功能，可以配置 Supabase 云存档。

---

## 步骤一：创建 Supabase 项目

1. 访问 [supabase.com](https://supabase.com) 并注册账号
2. 点击 "New Project" 创建新项目
3. 填写项目信息：
   - **Name**: 电子宠物（或任意名称）
   - **Database Password**: 设置一个数据库密码（请记住它）
   - **Region**: 选择离你最近的区域
4. 等待项目创建完成（约 2 分钟）

---

## 步骤二：获取项目 URL 和 Anon Key

1. 在项目左侧菜单中，点击 **Settings**（齿轮图标）→ **API**
2. 复制以下两个值：
   - **Project URL**（例如：`https://xxxxxx.supabase.co`）
   - **anon public** key（以 `eyJhbGciOi...` 开头的长字符串）

---

## 步骤三：配置游戏代码

打开 `game.js` 文件，找到文件开头的配置部分：

```javascript
// ===== 用户系统配置 =====
// Supabase 云存档配置（可选，不配置则使用本地多账号系统）
// 把下面两个值替换成你自己的 Supabase 项目信息即可启用云存档
const SUPABASE_URL = '';   // 例如：https://xxxx.supabase.co
const SUPABASE_ANON_KEY = ''; // 你的 anon public key
```

将你刚才复制的 URL 和 Anon Key 填入对应位置：

```javascript
const SUPABASE_URL = 'https://你的项目id.supabase.co';
const SUPABASE_ANON_KEY = '你的anon public key';
```

---

## 步骤四：创建 game_saves 数据表

1. 在 Supabase 左侧菜单中，点击 **Table Editor**（表格图标）
2. 点击 "Create a new table"
3. 按以下配置创建表：

| 字段名 | 类型 | 说明 |
|--------|------|------|
| `id` | `int8` | 主键，勾选 "Is Identity" |
| `user_id` | `uuid` | 用户 ID，勾选 "Not Null" |
| `game_data` | `jsonb` | 游戏数据 |
| `updated_at` | `timestamptz` | 更新时间，默认值：`now()` |

4. 设置主键：`id`
5. 添加唯一约束：`user_id`（每个用户只能有一条存档）

### SQL 方式创建（更快捷）

点击左侧菜单 **SQL Editor** → **New Query**，粘贴以下代码并运行：

```sql
create table game_saves (
  id bigint generated by default as identity primary key,
  user_id uuid references auth.users not null unique,
  game_data jsonb not null default '{}'::jsonb,
  updated_at timestamp with time zone default timezone('utc'::text, now()) not null
);

-- 开启行级安全（RLS）
alter table game_saves enable row level security;

-- 创建策略：用户只能读写自己的存档
create policy "Users can view their own save" on game_saves
  for select using (auth.uid() = user_id);

create policy "Users can insert their own save" on game_saves
  for insert with check (auth.uid() = user_id);

create policy "Users can update their own save" on game_saves
  for update using (auth.uid() = user_id);
```

点击 "Run" 执行 SQL。

## 常见问题

### Q: 不配置 Supabase 可以用吗？
A: 可以！游戏内置了**本地用户系统**，即使不配置 Supabase 也能：
- 注册/登录多个账号
- 每个账号独立保存游戏进度
- 自动登录

只是数据只存储在当前设备的浏览器中，无法跨设备同步。

### Q: 本地账号和云端账号有什么区别？
| 特性 | 本地账号 | 云端账号 |
|------|----------|----------|
| 注册登录 | ✅ | ✅ |
| 多账号切换 | ✅ | ✅ |
| 数据持久化 | ✅（浏览器本地） | ✅（云端） |
| 跨设备同步 | ❌ | ✅ |
| 清除浏览器数据 | 会丢失 | 不会丢失 |
| 需要配置 | 不需要 | 需要 |

### Q: 如何确认云存档是否生效？
A: 进入游戏后，点击右上角设置 ⚙️，查看"存档类型"：
- 📱 本地账号 = 仅本地存储
- ☁️ 云端存档 = 已连接 Supabase 云存档

### Q: 可以同时使用本地和云端账号吗？
A: 可以！游戏会自动根据配置选择最佳方案：
- 配置了 Supabase → 优先云端，失败时降级到本地
- 没配置 Supabase → 纯本地账号系统

---

## 数据安全说明

- **本地账号**：密码使用简单哈希存储，仅防肉眼查看，不具备真正的加密安全性
- **云端账号**：由 Supabase 提供专业的身份认证和数据加密
- 建议重要数据使用云端账号保存
